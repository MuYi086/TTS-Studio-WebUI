## 页面结构基线

当前页面是单页工作台，共 5 个主标签页，外加顶部全局工具栏。

### 1. 模型配置

包含两类配置：

- `LLM`（大语言模型）配置
  - 新增、编辑、删除配置
  - 字段：配置名称、模型名称、`Base URL`、`API Key`、额外参数 `JSON`
- `TTS` 配置
  - 新增、编辑、删除配置
  - 字段：配置名称、`Base URL`

行为基线：

- 页面会记住当前选中的 `LLM` 和 `TTS` 配置
- `LLM` 请求按 OpenAI 兼容 `chat/completions` 组织
- `TTS` 请求按现有 `/v2/synthesize` 协议组织

### 2. 音色资源库

包含两部分：

- 音色管理
  - 新增、编辑、删除音色
  - 上传参考音频
  - 试听参考音频
  - 字段：名称、描述、参考音频路径、`assetKey`
- 情绪描述预设
  - 新增、编辑、删除自定义情绪
  - 内置 8 个系统情绪
  - 每个情绪带 8 维向量
  - 支持启用 / 禁用

行为基线：

- 系统情绪默认存在：
  - 高兴、生气、伤心、害怕、厌恶、低落、惊喜、平静
- 角色音色分析和音色生成流程依赖该页的数据

### 3. 音效与滤波器

包含三类资源库：

- `SFX` 音效库
  - 新增、编辑、删除
  - 上传音频
  - 试听
  - 波形裁剪
  - 默认音量
  - 启用 / 禁用
- `BGM` 背景音乐库
  - 新增、编辑、删除
  - 上传音频
  - 试听
  - 波形裁剪
  - 默认音量
  - 启用 / 禁用
- 滤波器库
  - 新增、编辑、删除
  - 启用 / 禁用
  - 类型：
    - `lowpass`
    - `highpass`
    - `bandpass`
    - `distortion`

行为基线：

- 页面默认初始化 4 个滤波器：
  - 电话音
  - 水下
  - 老广播
  - 机械失真
- 音频裁剪使用 `trimStart/trimEnd`
- 试听进度条和裁剪可视化必须保留

### 4. 脚本制作

这是当前系统最复杂的页面，包含以下子域：

- 多脚本标签管理
  - 新建脚本
  - 切换脚本
  - 双击重命名
  - 删除脚本
- 角色侧栏
  - 新增角色
  - 编辑名称
  - 绑定音色
  - 设置角色音量
  - AI 分析音色
  - AI 生成音色
- 原文输入与分析
  - 输入小说 / 剧本文本
  - 选择 `LLM`
  - 执行 AI 深度分析
- 控制块插入
  - 插入 `dialogue`
  - 插入 `bgm`
  - 插入 `bgImage`
  - 设置背景图片块数量
- 配音与播放
  - 选择当前 `TTS`
  - 一键批量生成
  - 清空全部配音
  - 顺序播放 / 停止播放
- 脚本块编辑
  - `dialogue`
    - 角色、文本、情绪、强度、滤波器、音效、停顿、裁剪、音量、语速
    - 单行生成 / 清除 / 播放
  - `bgm`
    - 播放 / 停止
    - 选择 `BGM`
    - 设置块级音量
  - `bgImage`
    - 图片预览
    - 背景图片提示词
    - 复制提示词
    - 选择图片并保存
- 原始分析结果展示

行为基线：

- 脚本块是 3 类判别类型：
  - `dialogue`
  - `bgm`
  - `bgImage`
- 当前脚本切换前会先同步当前编辑状态
- 运行中禁止删除 / 切换脚本
- 顺序播放会处理 `BGM` 与背景图块的预扫描和切换

### 5. Prompt 管理

当前共有 3 组可配置模板：

1. 剧本拆分与分析 `Prompt`
2. 角色音色分析 `Prompt`
3. `Qwen` 音色生成参考文本模板

行为基线：

- 每组模板都支持：
  - 启用 / 禁用自定义
  - 保存
  - 恢复默认
- 各模板都有固定可用变量

## 顶部全局工具栏基线

当前全局工具栏提供以下能力：

- 导出完整工程
- 导入完整工程
- 导入 `TXT`
- 导出 `SRT`
- 导出音频
- 选择视频分辨率
- 生成视频

行为基线：

- 这些能力不依赖当前标签页，始终在顶部可用
- 导入 / 导出期间会显示禁用状态或处理中状态

## 核心流程基线

### 1. 本地工作流

- 页面启动后从 `localStorage` 恢复配置类数据
- 从 `IndexedDB` 恢复工程结构和二进制素材
- 自动把音色、音效、`BGM`、背景图、台词音频恢复到可播放状态

### 2. AI 分析工作流

- `LLM` 深度分析原文
- 产出角色、情绪、滤波器、`SFX`、`BGM`、背景图片提示词等结构化结果
- 写回脚本块与角色列表

### 3. 音色生成工作流

- 基于 `LLM` 生成角色音色描述
- 再调用音色设计服务生成参考音频
- 音频写入本地资产库并更新音色库与角色绑定

### 4. 台词生成工作流

- 生成前会确保参考音频已上传到 `TTS` 服务
- 单行台词生成后会把音频保存到 `IndexedDB`
- 批量生成会跳过已有音频

### 5. 播放与导出工作流

- 顺序播放会按脚本块顺序处理：
  - `bgm`
  - `bgImage`
  - `dialogue`
- 导出音频时要按相同时间轴混音
- 导出 `SRT` 与导出音频共享时间语义
- 生成 `MP4` 时叠加背景图与音频

- 可配置多组 `LLM` / `TTS`
- 可维护音色、音效、`BGM`、滤波器、情绪库
- 可创建多脚本标签并切换
- 可执行 AI 深度分析
- 可执行角色音色分析与音色生成
- 可单行生成、批量生成、清空音频
- 可顺序播放脚本
- 可导入 / 导出完整工程
- 可导出 `SRT`
- 可导出 `WAV`
- 可生成 `MP4`

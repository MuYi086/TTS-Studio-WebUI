# P0 手动回归入口

## 目的

`P0` 阶段不引入自动化测试。本文件定义重构期间固定使用的人工回归入口，确保每次迁移后都能用同一套样例检查关键链路。

## 固定工件

- 样例工程：`fixtures/p1-minimal-regression-project-v3.json`
- 结构校验脚本：`scripts/validate-p1-regression-fixture.mjs`
- 现有手动说明：`P1-最小回归验收.md`
- 当前行为对照页：`index.html`

## 回归前置

1. 先运行：

```bash
node scripts/validate-p1-regression-fixture.mjs
```

2. 确认样例工程结构合法后，再在浏览器打开当前旧页 `index.html`。

## 最小离线回归

### 1. 导入工程

- 点击“导入完整工程”
- 选择 `fixtures/p1-minimal-regression-project-v3.json`

### 2. 资源库恢复断言

导入后应至少看到：

- 音效库出现 `测试提示音`
- `BGM` 库出现 `测试BGM`
- 音色库出现 `旁白`
- 滤波器数量仍为 4
- 系统情绪数量仍为 8

### 3. 脚本恢复断言

导入后应满足：

- 当前脚本名为 `P1 最小回归验收工程`
- 一共 5 个块
- 顺序为：
  - `BGM play`
  - `bgImage`
  - `dialogue`
  - `dialogue`
  - `BGM stop`
- 两条台词都可直接播放
- 第一条台词包含 `测试提示音`
- 背景图块已经恢复图片，不是空白占位

### 4. 刷新恢复断言

- 刷新页面
- 上述资源库与脚本状态仍然存在

### 5. 导出链路断言

依次执行：

- 导出 `SRT`
- 导出音频
- 生成视频

关注点：

- 流程能跑通
- 不报致命错误
- 时间轴没有明显错乱

## 最小在线回归

在线回归依赖至少 1 个可用 `TTS` 配置。

### 1. 前置

- 选择一个可用 `TTS`

### 2. 单行重生成

- 清除第一条台词音频
- 重新生成第一条台词音频
- 确认新音频可以播放

### 3. 刷新恢复

- 刷新页面
- 确认刚生成的第一条台词音频仍可播放

### 4. 可选补充

如需要进一步确认任务链路，再执行：

- 一键生成配音
- 清空全部音频

## P0 完成标准

满足以下条件即可视为 `P0` 的回归入口已经建立：

1. 样例工程固定不变
2. 结构校验脚本固定可用
3. 旧页手动回归步骤固定可复用
4. 后续每个迁移阶段都能复用这套回归入口

## 使用原则

不是每次都要做全量深测，但只要动到了导入导出、本地恢复、脚本编辑、台词生成、顺序播放或视频导出，就必须回到这份入口重新核对。
